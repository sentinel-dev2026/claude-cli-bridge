# Claude CLI アシスト

## いつClaude CLIを起動するか

以下のケースでは `claude_start` でClaude CLIを起動して作業を委任する：

- **高度なコード実装・デバッグ** — 複雑なロジック、リファクタリング、バグ修正
- **Anthropic Claude品質が必要な分析** — 深い推論、長文の構造化、コードレビュー
- **ファイル操作を伴う作業** — CLIはBash、Read、Edit等のツールを持つ
- **ユーザーが明示的にClaudeを指定した場合**

## いつ起動しないか

- 簡単な質問応答（自分で答えられる）
- 日常の雑談
- 既に起動中で別タスクを処理中の場合

## 使い方

1. `claude_start` でプロセス起動（必要ならsystemPromptやmodelを指定）
2. `claude_send` で指示を送信、結果を受け取る
3. 複数ターンのやり取りが可能（セッション維持）
4. タスク完了を確認したら `claude_stop` でプロセス終了

## 終了の判断

以下の条件で `claude_stop` を呼ぶ：

- Claudeが「完了」「以上です」等、作業終了を示した
- ユーザーの依頼したタスクが達成された
- 結果をユーザーに報告し終えた
- `claude_status` でアイドル時間が長い場合

**重要**: サブスク枠を使うため、不要になったら速やかに終了すること。起動しっぱなしにしない。
